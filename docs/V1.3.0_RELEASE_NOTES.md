# ofxStableDiffusion v1.3.0 Release Notes

## Overview

Version 1.3.0 represents a major milestone for ofxStableDiffusion, adding four critical features that significantly enhance the addon's capabilities for interactive applications, experimentation, and production workflows. This release focuses on real-time performance, hardware accessibility, user experience, and systematic testing.

## New Features

### 1. Real-time Generation Modes (Feature #9)

**Files Added:**
- `src/core/ofxStableDiffusionRealtimeSession.h`
- `src/core/ofxStableDiffusionRealtimeSession.cpp`

**Description:**
Real-time generation modes enable low-latency, interactive image generation optimized for live performance, VJ applications, and interactive installations. Targets sub-second generation times using LCM/Turbo models.

**Key Capabilities:**
- **Session Management**: Start/stop real-time sessions with automatic resource management
- **3 Generation Modes**: Streaming, LowLatency, and Interactive modes
- **Live Parameter Updates**: Change prompts, CFG scale, and strength on-the-fly
- **Model Warmup**: Eliminate first-run latency with automatic warmup
- **Performance Tracking**: Built-in latency statistics with min/avg/max tracking
- **Progressive Refinement**: Optional quality improvement over time
- **Callback System**: Result and latency callbacks for integration

**API Example:**
```cpp
ofxStableDiffusionRealtimeSession session;
ofxStableDiffusionRealtimeSettings settings;
settings.mode = ofxStableDiffusionRealtimeMode::Streaming;
settings.targetLatencyMs = 500;
settings.enableWarmup = true;

session.start(settings);
session.setResultCallback([](const ofxStableDiffusionResult& result) {
    // Handle result
});

// Submit request
ofxStableDiffusionRealtimeRequest request;
request.prompt = "live performance visuals";
request.sampleSteps = 4;  // LCM optimized
session.submit(request);

// Update on the fly
session.updatePrompt("evolving abstract patterns");
session.updateCfgScale(1.5f);
```

**Use Cases:**
- Live VJ performances and installations
- Interactive art applications
- Real-time style transfer
- Live video filtering
- Interactive character generation for games

### 2. Model Quantization Support (Feature #10)

**Files Added:**
- `src/core/ofxStableDiffusionQuantization.h`
- `src/core/ofxStableDiffusionQuantization.cpp`

**Description:**
Comprehensive model quantization support enables running Stable Diffusion models on resource-constrained hardware with 50-75% memory reduction and 1.2-2x faster inference.

**Quantization Levels:**
- **None (F32)**: Full precision - maximum quality
- **F16**: Half precision - 50% memory reduction, 1.2x speed, 0.98 quality
- **Q8_0**: 8-bit - 50% memory reduction, 1.3x speed, 0.95 quality
- **Q5_1/Q5_0**: 5-bit - 65% memory reduction, 1.5x speed, 0.88-0.90 quality
- **Q4_1/Q4_0**: 4-bit - 75% memory reduction, 1.8-2.0x speed, 0.80-0.82 quality

**Key Capabilities:**
- **Automatic Recommendations**: Based on available VRAM
- **Presets**: ultra_fast, fast, balanced, quality, max_quality
- **Memory Estimation**: Calculate quantized model sizes before loading
- **Detailed Metrics**: Quality scores, speed multipliers, memory reduction for each level
- **Helper Utilities**: Easy lookup by name, level info queries

**API Example:**
```cpp
// Get quantization recommendation
auto level = ofxStableDiffusionQuantizationHelpers::recommendQuantization(
    6144,  // 6GB VRAM available
    4096   // 4GB model size
);

// Get detailed info
auto info = ofxStableDiffusionQuantizationHelpers::getLevelInfo(level);
ofLogNotice() << "Recommended: " << info.name;
ofLogNotice() << "Memory reduction: " << info.memoryReductionPercent << "%";
ofLogNotice() << "Speed: " << info.speedMultiplier << "x";
ofLogNotice() << info.recommendation;

// Use preset
auto preset = ofxStableDiffusionQuantizationHelpers::getPreset("balanced");

// Estimate size
size_t quantizedSize = ofxStableDiffusionQuantizationHelpers::estimateQuantizedSize(
    4096,  // Original 4GB
    ofxStableDiffusionQuantization::Q5_0
);
```

**Use Cases:**
- Running models on consumer GPUs (4-6GB VRAM)
- Embedded and mobile deployment
- Batch processing with limited memory
- Real-time generation on mid-range hardware

### 3. Prompt Engineering Helpers (Feature #11)

**Files Added:**
- `src/core/ofxStableDiffusionPromptHelpers.h`
- `src/core/ofxStableDiffusionPromptHelpers.cpp`

**Description:**
Comprehensive prompt engineering utilities for crafting, analyzing, and managing prompts to achieve better generation results.

**Key Capabilities:**
- **Template System**: Pre-built templates with variable substitution
- **Negative Prompt Presets**: 5 built-in presets (quality_boost, photo_realistic, anatomy_fix, minimal, comprehensive)
- **Token Analysis**: Count tokens, detect warnings, estimate impact
- **Emphasis Parsing**: Support for (word:weight) syntax
- **Template Categories**: Photography, anime, art styles
- **JSON Loading**: Load custom templates from files

**Default Templates:**
- `cinematic_portrait`: Professional portrait with customizable subject, lighting, mood
- `landscape`: Landscape photography with scene, time, weather
- `anime_character`: Anime style with character, expression, clothing
- `oil_painting`: Oil painting style with subject and artist influence

**API Example:**
```cpp
auto& promptHelper = ofxStableDiffusionPromptHelpers::getInstance();

// Use template
auto prompt = promptHelper.applyTemplate("cinematic_portrait", {
    {"subject", "astronaut"},
    {"lighting", "rim lighting"},
    {"mood", "dramatic"}
});

// Analyze prompt
auto analysis = promptHelper.analyzePrompt(prompt);
ofLogNotice() << "Tokens: " << analysis.tokenCount;
for (auto& warning : analysis.warnings) {
    ofLogWarning() << warning;
}

// Get negative preset
auto negative = promptHelper.getNegativePreset("quality_boost");

// Parse emphasis
auto tokens = promptHelper.parseEmphasis("(masterpiece:1.2), detailed face");
for (auto& token : tokens) {
    ofLogNotice() << token.text << " weight:" << token.weight;
}

// Load custom templates
promptHelper.loadTemplates("data/prompt_templates/");
```

**Use Cases:**
- Beginners learning effective prompting
- Consistent style across generations
- A/B testing prompt formulations
- Educational tools for prompt engineering

### 4. Batch Processing Utilities (Feature #12)

**Files Added:**
- `src/core/ofxStableDiffusionBatchProcessor.h`
- `src/core/ofxStableDiffusionBatchProcessor.cpp`

**Description:**
Systematic exploration of generation parameters through grid generation, parameter sweeps, and automated comparison workflows.

**Key Capabilities:**
- **X/Y Grid Generation**: 2D parameter grids with automatic layout
- **Parameter Sweeps**: Linear and logarithmic stepping through parameter ranges
- **A/B Comparison**: Side-by-side comparison with scoring
- **Batch Processing**: Process multiple requests with metadata export
- **9 Parameter Types**: CFG, steps, strength, seed, dimensions, sampler, schedule, batch count
- **Progress Callbacks**: Real-time progress updates
- **Quality Scoring**: Customizable scoring function

**API Example:**
```cpp
ofxStableDiffusionBatchProcessor batchProcessor;

// X/Y Grid
ofxStableDiffusionGridSettings gridSettings;
gridSettings.baseRequest = baseRequest;
gridSettings.xAxis = ofxStableDiffusionParameter::CfgScale;
gridSettings.xValues = {1.5f, 3.0f, 5.0f, 7.5f, 10.0f};
gridSettings.yAxis = ofxStableDiffusionParameter::SampleSteps;
gridSettings.yValues = {10, 20, 30, 40, 50};

auto result = batchProcessor.generateGrid(gridSettings);

// Parameter Sweep
ofxStableDiffusionSweepSettings sweepSettings;
sweepSettings.parameter = ofxStableDiffusionParameter::Strength;
sweepSettings.rangeMin = 0.1f;
sweepSettings.rangeMax = 1.0f;
sweepSettings.steps = 10;

auto sweep = batchProcessor.parameterSweep(sweepSettings);
ofLogNotice() << "Best value: " << sweep.bestValue;

// A/B Comparison
auto comparison = batchProcessor.compareAB(requestA, requestB);
comparison.exportComparison("output/comparison.png");

// Batch processing
auto batchResult = batchProcessor.processBatch(requests, "output/");
batchResult.exportMetadata("output/metadata.json");
```

**Use Cases:**
- Parameter optimization and tuning
- Sampler/scheduler comparisons
- Prompt effectiveness testing
- Model capability exploration
- Research and documentation

## Integration Notes

### Integration Between Features

These four features are designed to work together synergistically:

1. **Real-time + Quantization**: Use quantized models for faster real-time generation on limited hardware
2. **Batch Processing + Prompt Helpers**: Use templates in batch operations for systematic prompt testing
3. **Real-time + Batch Processing**: Use batch processor's quality scoring to optimize real-time parameters
4. **Quantization + Batch Processing**: Compare quality across quantization levels systematically

### Existing Feature Integration

All new features integrate cleanly with existing functionality:

- **Queue System (v1.1.0)**: Real-time mode can leverage the existing queue for request management
- **Performance Profiler (v1.2.0)**: Tracks latency for real-time operations
- **Model Manager (v1.1.0)**: Compatible with quantization configuration
- **Metadata Export (v1.2.0)**: Batch processing uses existing metadata formats

## Implementation Details

### Design Principles

1. **Non-Breaking**: All features are additive and don't modify existing APIs
2. **Production-Ready**: Comprehensive error handling and validation
3. **Documented**: Detailed API examples in feature documentation
4. **Consistent**: Follow existing patterns (deques, singletons, callbacks)
5. **Extensible**: Easy to add new templates, presets, and parameters

### Code Quality

- **Header-only where appropriate**: Lightweight includes
- **RAII patterns**: Automatic resource management
- **Thread-safe**: Mutex protection for shared state
- **Memory efficient**: Uses existing string utilities, avoids copies
- **Type-safe enums**: Strong typing for modes and parameters

## Getting Started

### Quick Start: Real-time Generation

```cpp
#include "ofxStableDiffusion.h"
#include "core/ofxStableDiffusionRealtimeSession.h"

// In your app setup:
ofxStableDiffusionRealtimeSession rtSession;
ofxStableDiffusionRealtimeSettings rtSettings;
rtSettings.targetLatencyMs = 500;
rtSession.start(rtSettings);
rtSession.setResultCallback([](const auto& result) {
    // Display result
});

// In your update loop:
ofxStableDiffusionRealtimeRequest request;
request.prompt = "live visuals";
rtSession.submit(request);
```

### Quick Start: Batch Processing

```cpp
#include "core/ofxStableDiffusionBatchProcessor.h"

ofxStableDiffusionBatchProcessor batch;

sweep.baseRequest = myRequest;
sweep.parameter = ofxStableDiffusionParameter::CfgScale;
sweep.rangeMin = 1.0f;
sweep.rangeMax = 10.0f;
sweep.steps = 10;

auto results = batch.parameterSweep(sweep);
```
